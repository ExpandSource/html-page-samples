1. 기획 및 발표 방식의 참신함
스팀(Steam) 테마 웹페이지:
프로젝트 웹페이지를 스팀 플랫폼 형태로 구현한 아이디어가 직관적이고 매우 참신했습니다.

시선을 사로잡는 영상 활용:
시청각 자료(영상)를 적극적으로 활용해 청중의 시선을 단번에 끄는 점이 신선했습니다. ('신창섭의 정상화' 밈)

대화형 발표 진행:
단순한 정보 전달을 넘어 대화 형식으로 진행된 발표 방식이 돋보였습니다. 마치 NotebookLM이 대화를 생성하며 학습하는 과정을 보는 듯해 아주 신선하고 흥미로웠습니다.



2. 문제 접근 및 데이터 분석의 논리성
핵심을 찌르는 문제 접근:
'이미 유명한 게임만 지속적으로 노출된다'는 기존 시스템의 한계를 짚어내고, 이를 해결하고자 한 문제 접근 방식이 훌륭했습니다.

타당한 데이터 전처리:
전처리 과정에 대한 설명이 명확했습니다. 특히, 데이터가 적게 쌓인 게임을 이상값(Outlier)으로 판단하여 과감히 제거한 기준과 논리가 타당했습니



3. 기술적 깊이와 실무적 공감대
쉬운 기술 용어 설명:
'Data Leakage(학습한 데이터를 그대로 테스트하여 성능이 과적합되는 오류)'라는 전문적인 개념을 청중이 이해하기 쉽게 잘 설명해 주어 좋았습니다.

실무를 관통하는 이상 탐지(품질 보정):
이상 탐지를 통한 리뷰 품질 보정 과정은 실무적으로 깊이 공감되는 부분이었습니다. 실제 제 경험상으로도 게임 업계에서 리뷰 어뷰징(주작)이 암암리에 이루어지고 있기 때문에, 이번 프로젝트의 분석 내용이 현실적으로 더욱 와닿았습니다. (암암리도 아님 그냥 단가가 적힌 제안서로 있음)


4. 모델링에 대한 깊은 고민과 기술적 문제 해결
흥행 예측 모델의 데이터 균형 확보: 흥행 예측 모델을 구축할 때 필연적으로 발생하는 데이터 불균형(Data Imbalance) 문제를 어떻게 해결하고 밸런스를 맞출지 깊이 있게 고민한 흔적이 잘 보여서 좋았습니다.

언어적 한계를 극복한 자연어 검색 시도:
스팀 자체 검색 기능의 고질적인 문제(한글과 영어 검색 결과가 별개로 도출되는 현상)를 정확히 짚어내고, 언어별 차이를 인식하여 이를 해결하려 한 자연어 검색 모델 기획이 매우 훌륭했습니다.

문맥(Context) 중심의 알고리즘 적용:
단순한 단어 매칭을 넘어, 머신러닝(ML)에는 TF-IDF를, 딥러닝(DL)에는 임베딩(Embedding) 기법을 적용한 점이 인상 깊었습니다. 표면적인 단어가 다르더라도 그 이면의 '뜻과 문맥'을 파악하여 검색과 감성 분석에 활용하고자 한 기술적 접근이 아주 좋았습니다.

기존에 생각했던 불편함이였지만, 문제점으로 인식하고 이를 머신러닝과 딥러닝으로 분석, 해결책으로 AI로 연결한 것이 창의력이 있었다고 생각합니다.
한 번쯤 생각해봤던 질문에 대해 다양한 모델들을 유연하게 적용해봤다는 것이 감명깊었다.
